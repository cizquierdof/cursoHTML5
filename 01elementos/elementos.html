Elementos HTML

Como ya dije en el capítulo anterior, los documentos html se hacen con elementos, los cuales consisten en una etiqueta de inicio, una etiqueta de final y el contenido que va en el medio, el elemento es todo; etiquetas y contenido. Por ejemplo, un elemento de párrafo sería:

<p>Esto es un párrafo</p>

  • <p> es la etiqueta de inicio
  • </p> es la etiqueta de final
  • Esto es un párrafo es el contenido

la etiqueta <p> le dice al navegador que lo que hay entre <p> y </p> tiene que tratarlo como un párrafo de texto.

Nota: No siempre hay etiquetas de cierre en los elementos, algunos solo tienen etiqueta de apertura, uuno de los casos más sencillos es el elemento salto de línea <br> no necesita final, cuando el navegador encuentra eta etiqueta salta a la línea siguiente.

Elementos anidados

Ya hemos visto que los elementos html pueden estar anidados, esto es elementos que contienen otros elementos. En el ejemplo:

<!DOCTYPE html>
<html>

<body>
<h1>My First Heading</h1>
<p>My first paragraph.</p>
</body>

</html>

El  elemento <body> contiene a los elementos <h1> y <p>, y el elemento <html> contiene a su vez el elemento <body>

Elementos vacíos

Algunos elementos html no llevan contenido, por eso se llaman elementos vacíos Un ejemplo es otra vez la etiqueta <br> que define un elemento de salto de línea, no tiene contenido, solo le indica al navegador que debe saltar a la línea siguiente.

Comentarios de código

Cuando la cantidad de código empieza a crecer, es muy importante comentar cada parte importante del código que escribimos. Todos los lenguajes de programación disponen de alguna forma de hacer esto; en HTML se utiliza una etiqueta especial <!-- para empezar el comentario y --> para terminarlo. Ejemplo:

<!-- Esto es un comentario -->

El navegador no muestra los comentarios, pero ayudan a documentar el HTML añadiendo notificaciones y recordatorios. Los comentarios también son excelentes para corregir errores. Marcar parte del código como comentario permite ver que ocurre cuando deja de ejecutarse ese trozo.

<!-- NO EJECUTAR EL SIGUIENTE CODIGO
      <img border="0" src="una_imagen.jpg">
-->

También se pueden utilizar comentarios condicionales haciendo que las líneas de programa solo se ejecuten si se cumple una condición.

<!-- if IE 8] --> Este código solo se ejecuta en Internet Explorer<![endif]-->

También puede haber comentarios generados por programas, por ejemplo Dreamweaver utiliza los comentarios <!-- InstanceBegin... y <!-- InstanceEnd... para marcar las diferentes secciones de sus plantillas.

nota: La mayoría de editores utilizan una codificación por colores para los distintos tipos de texto que se usan, así por defecto en Dreamweaver se usa el negro para texto normal, azul para palabras reservadas y etiquetas, gris para los comentarios, etc. por lo que es más fácil reconocerlos entre las otras posibles líneas de código.

Escribir etiquetas con minúsculas

Esto es una recomendación, no una obligación. Las etiquetas html no distinguen entre mayúsculas y minúsculas, sin embargo W3C la entidad que recopila y sostiene los estandares HTML, CSS y otros, recomienda usar minúsculas en HTML 5, en XHTML una versión más estricta del HTML, si las exige. El uso de mayúsculas puede considerarse una mala práctica.
