Curso de HTML5

Estilos

HTML sirve para dar estructura al documento web, pero aparte de unos pocos efectos de formateo como son los caracteres en negrita, itálica, encabezados, etc. a base de etiquetas no conseguiremos resultados impactantes. Los colores, imágenes de fondo, los tipos de letra, en definitiva casi todo lo que hace que una página sea atractiva se consigue mediante estilos CSS. Estas siglas quieren decir Cascade Style Sheets, o en castellano, Hojas de Estilo en Cascada. El nombre hace referencia a que se van aplicando estilos en cascada, es decir unos encima de otros.

Los estilos tienen un orden jerárquico, se aplican según su importancia, origen, especificidad y orden de aparición, es decir que según donde estén escritos y que tipo de regla se aplique se aplicará uno u otro estilo. También se tiene en cuenta el tipo de dispositivo en el que se va a mostrar el documento. El orden que sigue un navegador para aplicar las reglas es muy complejo y sería algo a tratar en un curso de CSS, por tanto no voy a meterme con ello en profundidad, solo decir que es de extrema importancia tener claro dicho orden para no volverse loco cuando más tarde empieces a aplicar estilos complejos.

A modo de resumen el orden es:

  1. Tipo de medio: pantalla, móvil, tablet, etc. Esto se especifica al definir los estilos mediante declaraciones.
  2. Importancia y origen de las reglas CSS. A su vez tienen este orden:
    1. Reglas por defecto del navegador, cada navegador tiene las suyas propias.
    2. Preferencias normales del usuario.
    3. Reglas normales declaradas en los estilos del autor.
    4. Reglas declaradas !important por el autor.
    5. Preferencias con !important del usuario.
  3. Especificidad del selector. Por ejemplo una etiqueta es menos especifica que una class.
  4. Orden de aparición de la regla.

Formas de aplicar estilos CSS

Hay tres formas de ir declarando estilos CSS, Inline o en línea, mediante la etiqueta <style> y declaraciones en un fichero de estilos aparte.

Estilos Inline

Este es el método más inmediato, consiste en añadir un atributo style dentro de la etiqueta HTML, por ejemplo para cambiar el color del texto en un encabezado:

<h1 style=”color:blue”>Este es un encabezado azul</h1>

El atributo style tiene la siguiente sintaxis:

style="propiedad:valor"

Se pueden añadir varias propiedades al mismo tiempo separandolas con ; por ejemplo:

<h1 style=”color:blue; font-style:italic”>Este es un encabezado azul</h1>

Esta forma está totalmente desaconsejada. ¿Ves cual es el fallo?

Cuando empieza a crecer el código de la página, aplicar estilos elemento por elemento es demencial, porque su mantenimiento resulta muy difícil. Imagina ir línea por línea para cambiar todos los estilos, de todos los elementos de la página.

Entonces si no se deben utilizar ¿Para que están?

Que estén desaconsejados no significa que no puedan resultar útiles en determinados casos. Supón que tienes aplicado un estilo para los enlaces de tu tira de navegación. Todos los enlaces se representarán según lo que indique la hoja de estilos, sin embargo te interesa que uno de ellos, y solo uno, tenga un estilo diferente. Aplicarle un estilo inline soluciona el problema, pero ojo; esto se debe hacer para este tipo de excepciones; si al final resulta que esto lo vas a repetir en más sitios, considera el uso de uno de los otros dos sistemas.

Estilos internos

Consiste en definir todos los estilos que se van a utilizar en el documento en único sitio, normalmente dentro de la sección <head> del documento. El navegador reconoce una definición de estilos todo lo que encuentra entre las etiquetas <style> y </style>. Por ejemplo queremos una página con fondo gris, texto de párrafo cían y encabezados amarillos, para hacerlo según el modelo de estilos internos pondriamos.

<!DOCTYPE HTML>
<HTML>
	<head>
		<meta charset="UTF-8" />
		<title>sample page</title>
		<style>
			body{
				background-color: grey;		
			}
			h1 {color: blue;}
			p {color: cyan;}
		</style>
	</head>
	<body>
		<h1>Encabezado en azul</h1>
		<p>Párrafo de texto en color cían</p>
	</body>
</HTML>

Este sistema es aconsejable cuando tienes que aplicar estilo a una sola página, si lo que quieres es que tu web, compuesta por muchas páginas, tenga un estilo uniforme es mejor utilizar el método que se explica a continuación.

Estilos externos

Es parecido a la aplicación de los estilos internos, ya que se hace la definición de todos los atributos en el mismo sitio; solo que esta vez se hace en un fichero aparte. Este fichero debe tener la extensión .css y es especialmente útil para dar un aspecto homogeneo a un grupo de páginas, de esta forma solo tenemos que indicar cual o cuales ficheros de estilo aplicar. Sobre todo, la enorme ventaja de esto es, que si queremos modificar el estilo, lo hacemos en único lugar y se aplicará en todas las páginas que lo carguen. Este sistema tiene, además la ventaja de separar totalmente los códigos, de manera que cuando escribimos HTML, nos centramos solo en la estructura, y cuando escribimos CSS nos centramos solo en la forma.

Los estilos externos CSS se asocian a una página HTML a través de una etiqueta <link> en la sección <head> de la siguiente forma: 

<!DOCTYPE HTML>
 <HTML>
 	<head>
         <link href ="/path/stylesheet.css" rel="styles.css">
     </head>

Donde stylesheet.css es el fichero de estilo CSS y /path/ es el directorio donde se encuentra, pudiendo estar en la misma carpeta que el documento html o no. Por ejemplo si el archivo es miEstilo.css podemos encontrarnos:

  • href=miEstilo.css el archivo está en el mismo directorio que la página
  • href=../stiles/mi-estilo.css es una dirección relativa. Para encontrar el fichero nos iremos una carpeta hacia arriba y dentro de esta en otra llamada stiles.
  • href=/dir_raiz/estilos/miEstilo.css es una dirección absoluta, el fichero está en la carpeta estilos que cuelga del directorio raíz del sitio.

Tanto en las hojas de estilo internas como en las externas se pueden definir varios atributos a la vez para un mismo elemento separándolos con punto y coma, la sintaxis es la siguiente:

elemento { atributo1:valor; atributo2:valor;...atributo:valor;}

también se pueden definir grupos de atributos para varios elementos a la vez separando cada elemento por comas:

elemento1, elemento2, elemento3 {atributo1:valor; atributo2:valor;...}

en la siguiente línea vamos a aplicar un estilo de fuente y color simultáneamente a las cabeceras h1, h3 y a los párrafos de texto.

h1, h3, p {color:#0000cc; font-style:italic; }

Herencia de estilos

Quizá me esté metiendo en un jardín del que me resulte complicado salir, pero me gustaría hablar un poco de la herencia de estilos, ya que creo que es uno de los aspectos de CSS más difíciles de pillar. Algo importante a entender es, que pasa si le aplicamos estilo a un elemento, que contiene otro elemento, que ya tenía un estilo.

Todo esto tiene que ver con lo que hablamos al principio del tema sobre el orden de aplicación del CSS. Vamos a verlo con un ejemplo. He intentado que sea lo más sencillo posible. Mira con atención el siguiente documento html.

<!doctype html>
<html>
<head>
<meta charset="utf-8">
<title>Herencia de estilos</title>

<style>
   h1 {color:orange; font-size:36px;}
   header {color:blue; font-size:24px;}
   article {font-size:9px;}
   article h1 {color:black; font-size:18px;}
</style>

</head>

<body>
<h1>TITULO INDEPENDIENTE CON</h1>
<p>Este es un pequeño párrafo independiente. Todos los títulos y párrafos 
		de esta página están construidos con &lt;h1&gt; y &lt;p&gt; respectivamente.</p>

<header>
<h1>TITULO DE ENCABEZADO</h1>
<p>Un pequeño párrafo para el encabezado</p>
</header>

<article>
<h1>TITULO DE <span style="color:red;">ARTICULO</span></h1>
<p>Otro pequeño párrafo para el artículo</p>
</article>

</body>
</html>

Lo primero que vemos es que le he aplicado una hoja de estilos interna con la etiqueta <style> en la que sucesivamente se hace lo siguiente:

  • Los encabezados <h1> tendrán texto naranja de 36 píxeles.
  • Las cabeceras <header> tendrán texto azul de 24 píxeles.
  • Los artículos <article> tendrán texto de 9 píxeles.
  • Los encabezados de los artículos serán de texto negro y 18 píxeles.
  • El elemento <spam> dentro del<h1> del artículo tiene un estilo inline que le cambia el color a rojo.

Veamos que es lo que ocurre:

Efectivamente los encabezados son naranja y de 36px salvo el del artículo.

Luego vemos que se aplica texto azul de 24px en la sección de encabezado, pero solo al texto del párrafo ¿porqué?

Pues porque solo hemos dicho de cambiar el color a <header> y la regla que cambia el formato a los <h1> es más específica y sobrescribe a la más genérica del <header>.

Por último tenemos que para el artículo se cambia tanto el tamaño del texto general como el de <h1> que, además también cambia de color, y eso porque también se aplica una regla específica para los <h1> del <article>. Date cuenta de que en el párrafo del artículo no se hereda el color de texto del párrafo de <header> ya que están en elementos completamente diferentes.

¿Y que pasa si no aplicamos la primera regla sobre los h1? Pues que el encabezado de <header> no hereda esta regla, aunque en realidad si que hereda una, la regla genérica del navegador que estés utilizando, pero eso ya queda para un curso de CSS. Pulsa el botón para ver la página anterior sin regla general para <h1>.

Lo dicho, en este caso tanto los encabezados <h1> y los párrafos heredan el color y el tamaño de texto especificados en las reglas aplicadas al elemento que los contiene, respectivamente <header> y <article>.

Si ya sé que según lo anterior tanto los párrafos como los textos deberían tener el mismo tamaño en <header> o en <article>, y es evidente que esto no es así; pero esto es porque la propia etiqueta <h1> modifica el aspecto del texto. osea que aunque en <header> se aplique font-size=24px <h1> es mucho mayor, porque el tamaño de fuente de los encabezados es relativo a la font-size.

Por último habrás comprobado que el estilo inline de <spam> machaca a todos los demás.

Bien, no se si he conseguido explicar este tema, pero por lo menos espero haberte transmitido la imperiosa necesidad de manejar cómodamente la herencia de estilos CSS, porque si no, ¡estás listo!

Atributos de estilo

Fuentes CSS

Algunas de las propiedades que se pueden definir para las fuentes de texto con CSS son:

  • La propiedad color indica el color del texto de un elemento HTML
  • La propiedad font-family indica la font que se utilizará.
  • Se utiliza font-size para indicar el tamaño del texto.
  • Con font-style se puede elegir si queremos el texto normal, en negrita, itálica, etc.
  • text-align indica el tipo de justificación del texto; derecha, izquierda, centrado,..

En el siguiente ejemplo se va a aplicar la font verdana, con un tamaño del 300%, centrado y color azul a una cabecera h1, y de forma similar aplicaremos color rojo, font courier y tamaño del 160% a un párrafo de texto:

<!DOCTYPE HTML>
<HTML>
<head>
<meta charset="utf-8">
<style>
   h1 {
       font-family:verdana;
       font-size:300%;
		text-align: center;
		color: #0000FF;
   }
   p {
		color: #FF0000;
		font-family: courier;
		font-size:160%;
	} 
   </style>
</head>
<body>
     <h1>Esto es un encabezado azul, fuente Verdana y tamaño 300%</h1>
     <p>Esto es un párrafo color rojo, courier y tamaño 160%</p>
</body>
</HTML>

El modelo de cajas de CSS

Como ya vimos en el capitulo de introducción, cada elemento HTML visible está contenido dentro de una caja aunque normalmente no pueda verse. Para poder visualizar esta caja se puede utilizar el atributo border para dibujar una línea alrededor del elemento. A estas alturas he de suponer que ya estás introduciendo código en un editor y comprobando los resultados, así que prueba lo siguiente:

<!DOCTYPE HTML>
<html>
	<head>
		<meta charset="UTF-8" />
		<title>sample page</title>
		<style>
			h1 {border:solid black;}
			p {border: dashed grey;}
		</style>
	</head>
	<body>
		<h1>Encabezado encuadrado</h1>
		<p>Párrafo de texto encuadrado con una linea de guiones</p>
	</body>
</html>

Verás que el resultado es un encabezado enmarcado por una linea negra continua y a continuación un párrafo enmarcado por una linea de guiones gris. Esta es la caja que encierra los elementos HTML y que además tiene otros dos atributos importantes, el padding o espacio interior del borde que es el espacio que separa el borde de los elementos contenidos en la caja; y el margin o margen que es el espacio desde el borde de la caja hasta otros elementos contiguos del documento. Estos conceptos son otros para los que también es obligatorio comprender muy bien si luego queremos hacer correctamente el formato de la página. Veamos un ejemplo:

<!DOCTYPE HTML>
<html>
	<head>
		<meta charset="UTF-8" />
		<title>sample page</title>
		<style>
			h1 {
				background-color:#c0c0c0;
				border:solid black;
				padding: 5px; margin: 10px;
			}
			p {
				background-color:#ffff80;
				border: dashed grey;
			}
		</style>
	</head>
	<body>
		<h1>Encabezado encuadrado</h1>
		<p style="padding: 5px; margin: 20px;">Esto es un párrafo</p>
		<p style="padding: 10px; margin: 50px;">Esto es otro parrafo con más margen
 y padding</p>
	</body>
</html>

Como puede verse en el resultado del ejemplo anterior, el segundo párrafo es más ancho y esta más separado de los extremos de la página que el primero. Es importante entender que el tamaño de un elemento es la suma contenido+padding+border+margin donde el tamaño del contenido es el del texto, imagen u objeto que contenga, o si se ha definido su ancho y alto.
