Curso de HTML5

Imágenes

Despues de los textos, lo más utilizado en las páginas web son probablemente las imágenes y como es de suponer hay una etiqueta específica para ellas <img>, su sintaxis es la siguiente:

<img src="url" alt="texto">

La etiqueta <img> está vacía, solo contiene atributos y tampoco tiene etiqueta de cierre. de los atributos que puede llevar el único imprescindible es src que es el que le indica al navegador donde debe buscar la imagen a representar. Este atributo es una URL, si se indica solo el nombre del fichero de la imagen p.e: miImagen.jpg, esta se busca en el mismo directorio donde este la página que lo llama. Si el archivo no está en el mismo directorio deberemos poner la ruta hasta él. La ruta podrá ser absoluta o relativa.

<img src="miImagen.jpg">

el fichero debe estar en el mismo directorio.

<img src="/images/miImagen.jpg">

el fichero estará en una subcarpeta llamada images.

<img src="http://www.miPagina.com/images/miImagen.jpg">

en este caso la imagen se busca en la capeta images del sitio www.miPagina.com.

Si no se encuentra la imagen, entonces el navegador muestra un icono de enlace roto.

Atributo alt

Un segundo atributo importante relacionado con las imágenes es alt. El texto que se defina dentro de este atributo es muy útil ya que si no se encuentra la imagen el navegador pondrá el texto alternativo en su lugar, también lo utilizan los lectores de pantalla para personas con visión reducida ya que es este texto el que 'leen' cuando encuentran una imagen en una página web.

Este atributo es un requerimiento en HTML5, se puede poner img sin él y funcionará en los navegadores, pero se considera una mala práctica y la página no será validada correctamente.

Tamaño de la imagen

Siempre que se carga una imagen se debería poner el tamaño al que queremos representarla, y esto por varias razones, para poder controlar como se va a ver finalmente y también porque si el tamaño de la imagen es desconocido la página puede parpadear mientras está cargándola.

Para poner el tamaño de una imagen en la etiqueta se puede hacer por dos medios, el antiguo que consiste en utilizar los atributos de ancho y alto respectivamente width y height. la siguiente imagen se representará con 128 píxeles de ancho y 128 de alto:

<img src="miImagen.jpg" alt="algún texto" width="128" height="128">

Pero también se puede hacer lo mismo mediante el atributo style:

<img src="miImagen.jpg" alt="algún texto" style="width:128px;height=128px">

Ambas expresiones hacen exactamente lo mismo. Observar que los atributos width y height no indican que son píxeles mientras que en style sí; esto es porque con style no se está limitado a píxeles, también se puede poner porcentajes respecto del contenedor.

El siguiente código aplica todo lo dicho anteriormente, pincha el botón para ver los resultados.

<!doctype html>
  <html>
  <head>
  	<meta charset="utf-8">
  	<title>Documento sin título</title>
  </head>
	 <body>
  		<h3>Una imagen normal</h3>
  		<img src="images/emoticono1.png" alt="Emoticono 1">
  		<h3>La misma imagen con ancho y alto definidos</h3>
  		<img src="images/emoticono1.png" alt="Emoticono 1" 
				style="width:256px;height:256px">
  		<h3>Ahora una imagen no encontrada sin atributo alt</h3>
  		<img src="fakeimg.jpg" >
  		<h3>Otra imagen no encontrada pero con alt</h3>
  		<img src="fakeimg.jpg" alt="Esta imagen no se ha encontrado">
 	 </body>
  </html>

¿Ancho y alto o style? Aunque la flexibilidad que supone el poder poner porcentajes además de píxeles ya indicaría que es mejor utilizar style también hay otra razón, y es evitar que el uso de hojas de estilo puedan cambiar el tamaño que queremos para la imagen.

Ni que decir tiene que con el atributo style se puede cambiar también el resto de estilos aplicables a una imagen, tales como añadirle un borde, posicionamiento en la página, añadirle un float, etc.

Uso de imágenes como enlaces

En el capítulo de enlaces ya se habló de como utilizar las imágenes como enlaces. También hay otra forma muy común de utilizar la imágenes como enlaces y es mediante el mapeo. Esto consiste en definir regiones de una imagen que al pulsar sobre ellas redirigen a un enlace. Veamos esto con algo de código:

<!doctype html>
<html>
<head>
<meta charset="utf-8">
<title>Documento sin título</title>
</head>
<body>
	<h1>Imagen mapeada</h1>
	<img src="images/emoticono2.png" alt="" style="width:400px;height:400" 
		usemap="#Map"/>
 	<map name="Map">
    	<area shape="rect" coords="77,99,320,207" href="ojos.html" 
			target="_self" alt="Ojos">
    	<area shape="rect" coords="107,213,285,372" href="boca.html" 
			target="_self" alt="Boca">
  	</map>
</body>
</html>

La etiqueta figure

Aunque lo veremos más tarde en la parte específica de HTML5, no quiero terminal el capítulo sin hablar un poco de la etiqueta <figure>. Esta etiqueta es nueva, y su objetivo es más semántico que funcional. Es decir <figure> no sustituye a <img>, para mostrar una imagen en la página se va a tener que seguir utilizando esta, pero si la colocamos dentro del contenedor <figure> le vamos a indicar a los navegadores y buscadores que lo que contiene es eso, una figura lo cual redunda en una mejor indexación de la página además de que la estructura queda más clara, al tratar con <figure> en vez de <img> se puede diferenciar más fácilmente lo que son imágenes, por ejemplo de un artículo, de las imágenes de otro tipo, como las de un banner y darles diferente tratamiento mediante CSS. Hay que apuntar también que <figure> no esta pensada solo para imágenes, también es útil aplicándola a gráficos, diagramas, listados de código, etc. Su uso vendría a ser algo así:

<figure>
<img src="miImagen.jpg" alt="texto alternativo" width="110" height="80">
<figcaption>Texto asociado</figcaption>
</figure>

El elemento <figcaption> hace referencia a los textos que suelen tener las figuras en el pié y que se utilizan como título o breve explicación del contenido. También es una etiqueta pensada para estructurar, entre otras cosas nos permitirá fácilmente dar el mismo aspecto a todas las cabeceras de foto de uno o varios artículos. El siguiente código CSS, (ya vimos un poco de esto en el capítulo de Estilos), formatea todas las cabeceras de las imágenes de todos los artículos como texto en negrita y centrados:

article figure figcaption	{
 		font-weight: bold;
 		text-align: center;
	}
